# 完整示例

> 用一本假想的星际末世小说"星河纪元"演示整个工作流程。

---

## 示例设定

- **书名**：星河纪元
- **题材**：星际末世科幻
- **叙事框架**：三段式引擎
- **自主程度**：L2 半自动

---

## 第一步：初始化

用户对 Agent 说：

> "你好，我的小说Agent，我的新书叫《星河纪元》，末世科幻题材，请帮我初始化。"

Agent 自动执行：
1. 创建 `world.md`、`characters/`、`outline/`、`beats/`、`chapters/` 等目录结构
2. 基于题材生成 `world_template.md` 的框架
3. 生成 `style_guide.md`
4. 提示用户填写主角和主要角色设定

---

## 第二步：填定设定

### world.md（节选）

```markdown
# 世界底层规则

## 地理/世界构造
- 旧地球：末世前的文明遗址，被丧尸（代号"虫群"）占据
- 星际殖民站"方舟"：人类最后的据点，位于月球背面
- 漂移区：地球与月球之间的废弃空间站群，是黑市和赏金猎人的地盘

## 力量体系
- 神经增幅（NA）：通过义体改造增强战斗能力，分 C/B/A/S 四级
- 代价：每次增幅升级后有 10% 概率失去部分记忆
- 觉醒者：极少数人出现异能，概率约 1/10000

## 主要势力
- 方舟议会：控制月球基地，贵族阶层
- 漂移区商会：中立，黑市贸易
- 虫群：不是单一势力，是地球上的丧尸集合体
```

### characters/protagonist.md

```markdown
# 陆辰

## 基本信息
- 年龄：28岁
- 性别：男
- 外貌特征：178cm，义体右臂（军用级），银灰色短发，
  总是穿着带有方舟议会徽章的旧军装（左袖管有烧灼痕迹）

## 性格
关键词：冷静、实用主义、底线感强
说明：前方舟特种兵，因"记忆门"事件被驱逐，现在漂移区做赏金猎人
```

---

## 第三步：生成大纲

用户说：

> "帮我生成一个大纲，三段式，分四卷"

Agent 运行：

```bash
python scripts/outline_generator.py ./星河纪元 三段式 4
```

输出：

```
✅ 卷1大纲已生成: outline/volumes/volume_01.md
✅ 卷2大纲已生成: outline/volumes/volume_02.md
✅ 卷3大纲已生成: outline/volumes/volume_03.md
✅ 卷4大纲已生成: outline/volumes/volume_04.md
```

用户根据输出的大纲模板，填入具体内容：

```markdown
## 卷一：黑市之星

**核心主题**：生存与人性
**章节范围**：ch001 - ch030
**核心人物**：陆辰、苏九（漂移区商人）、林博士（方舟科学家）
**核心冲突**：陆辰发现方舟议会在秘密用活人做NA增幅实验

### 开局（ch001-ch005）
| 章节 | 核心事件 |
|------|---------|
| ch001 | 陆辰在漂移区接了一单"运输"任务 |
| ch002 | 货物是三名被麻醉的方舟平民 |
| ch003 | 雇主露出真目 |
| ... | ... |
```

---

## 第四步：写第一章

用户说：

> "写第一章"

Agent 读取：
- `world.md`（世界观）
- `characters/protagonist.md`（主角设定）
- `outline/volumes/volume_01.md`（卷大纲）
- `beats/TRACKING.md`（伏笔状态）
- `style_guide.md`（写作风格）

Agent 写作流程：

```
1. 章节规划：
   起：陆辰在漂移区醒来，接到任务
   承：接下运输任务，发现货物异常
   转：雇主是方舟议会的秘密部队
   合：结尾钩子——"他说：那三个人，不是货物"

2. 写草稿（3000+字）

3. 自审：
   ✅ 字数达标（3200字）
   ✅ 承接上章（无上章，此为开局）
   ✅ 对话有穿插
   ✅ 动作/心理描写充足
   ✅ 结尾有钩子

4. Editor Agent 审稿：
   ✅ 无错别字
   ✅ 设定一致（NA增幅描述与world.md一致）
   ✅ 伏笔 B001 已埋入
   ✅ 文笔正常

5. 写入 chapters/ch001.md

6. 更新 beats/TRACKING.md：
   新增 B001 | 伏笔：方舟议会在做人体实验 | ch001 | ch020 | pending

7. 通知用户（飞书推送）：
   "第1章《黑市之星》已完成，字数：3200字"
```

---

## 第五步：一致性检查

用户说：

> "检查一下前五章有没有矛盾"

Agent 运行：

```bash
python scripts/consistency_check.py ./星河纪元
```

输出：

```
==================================================
小说 Agent 一致性检查
检查路径: ./星河纪元
==================================================

发现 2 个潜在问题：

[WARNING] 角色陆辰的"义体右臂"描述在 ch003 和 ch005 中不一致：
  ch003 描述为"军用级义体"
  ch005 描述为"工业级义体" | characters/protagonist.md

[WARNING] 道具"NA增幅剂"的持续时间矛盾：
  world.md 定义为"4小时"
  inventory/equipment.md 中为"6小时"
```

Agent 提示用户：

> "发现两处矛盾，请确认：
> 1. 陆辰的义体级别是军用级还是工业级？
> 2. NA增幅剂的持续时间是4小时还是6小时？"

用户确认后，Agent 自动修正。

---

## 完整工作流回顾

```
用户指令
    ↓
意图识别（是"检查伏笔"）
    ↓
Supervisor 分发 → Editor Agent
    ↓
Editor Agent 调用 beat_tracker.py
    ↓
发现问题 → 上报用户
    ↓
用户确认 → Agent 更新 beats/TRACKING.md
    ↓
用户收到通知
```

这就是小说 Agent 的典型工作流程。
