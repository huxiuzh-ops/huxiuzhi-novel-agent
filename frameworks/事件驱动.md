# 事件驱动

> 以关键事件为节点的叙事框架。适合快节奏、高潮迭起、爽文型作品。

---

## 核心理念

**事件 → 推进 → 新事件 → 再推进**

不追求细水长流，不铺垫冗长的日常，用**关键事件**来切割叙事，每个事件有明确的前因后果。事件之间可以跳跃，不强调连续性，强调"精彩"。

---

## 什么是"事件"

一个完整的事件包含：

```
事件 = 触发（导火索）+ 发展（过程）+ 高潮（爆点）+ 后果（影响）
```

**好的事件例子**：
- 主角在黑市发现有人在贩卖异能者活体器官（触发）
- 追踪线索发现背后是联盟高层（发展）
- 双方正面冲突，异能失控（高潮）
- 主角被通缉，只能转入地下（后果）

**坏的事件例子**：
- 主角早上起床，吃了早饭，出门
- （这不叫事件，叫流水账）

---

## 事件驱动的节奏

### 事件密度

**核心原则**：每个事件要有意义（推进剧情 or 揭示真相 or 改变关系）

| 类型 | 事件密度 | 适用场景 |
|------|---------|---------|
| 高密度 | 每章1-2个事件 | 快节奏爽文、高潮迭起型 |
| 中密度 | 每5-10章1个核心事件 | 标准网文 |
| 低密度 | 每15-20章1个核心事件 | 日常为主、情感为主 |

---

## 事件的类型

### 1. 主线事件（推动核心剧情）

```
特点：必须有，直接影响核心冲突
示例：决战、重大发现、核心人物死亡或加入
节奏：每卷至少2-3个
```

### 2. 支线事件（丰富世界观、塑造人物）

```
特点：可选择，不影响主线
示例：配角的故事、番外型事件
节奏：穿插在主线之间，作为调剂
```

### 3. 钩子事件（留下悬念）

```
特点：小而精，核心目的是让读者追更
示例：一个未解之谜、一句意味深长的话、一个意外的相遇
节奏：每章结尾至少1个
```

## 事件设计技巧

### 事件之间的串联

事件之间要有逻辑关系，不能孤立存在：

```
事件A的后果 → 触发事件B
事件A揭示的信息 → 让事件B的冲突升级
```

**好的串联**：
> ch010：主角发现内鬼但没抓住（事件A后果）
> ch011：内鬼反过来诬陷主角（事件B，冲突升级）
> ch012：主角需要自证清白，同时追查真相（事件A揭示的信息触发新行动）

**坏的串联**：
> ch010：主角打了一架
> ch011：完全无关的另一场打架

---

### 事件的"代价"设计

每个事件发生后，主角必须付出代价或获得收益：

| 事件结果 | 代价/收益 |
|---------|---------|
| 胜利 | 收益明显，但埋下新隐患 |
| 失败 | 代价明显，但可能有意外收获 |
| 平局 | 双方各付出一定代价 |

**禁忌**：赢了就是赢了，没有任何后续影响

---

### 事件的高潮设计

每个事件都要有高潮点，即使是小事件：

```
小事件高潮：可以是情绪上的爆点（一个意外的拥抱/一句话的点醒）
大事件高潮：必须是外在冲突的爆发（打斗/谈判破裂/真相揭露）
```

---

## 事件驱动的章节结构

```
章节 = 事件开头（触发）+ 事件发展 + 事件高潮 + 章节钩子

- 开头：直接进入事件，不要从日常开始
- 发展：事件推进，可以有一两个小波折
- 高潮：本章的核心爆点
- 钩子：指向下一个事件
```

---

## 事件驱动检查清单

### 每章检查
- [ ] 本章有明确的事件吗？（不是流水账）
- [ ] 事件有触发、发展、高潮三个阶段吗？
- [ ] 事件发生后主角有代价或收益吗？
- [ ] 事件之间有串联吗？（不是孤立存在）
- [ ] 章节结尾有钩子吗？

### 每卷检查
- [ ] 核心主线事件有几个？（建议2-3个）
- [ ] 有没有支线事件作为调剂？
- [ ] 事件密度是否符合预期？
- [ ] 最后一个事件是否为下卷埋下伏笔？

---

## 适合题材

✅ 玄幻升级、打怪升级、爽文、系统流、快节奏悬疑、复仇文

❌ 不适合：情感细腻的纯爱、文学性强的作品、日常向作品

---

## 事件驱动 vs 三段式

| 维度 | 事件驱动 | 三段式 |
|------|---------|--------|
| 切割方式 | 事件为节点 | 节奏为节点 |
| 章节连贯性 | 可以跳跃，不强调连续 | 强调起承转合的连贯 |
| 日常戏份 | 少，事件之间快速切换 | 多，靠日常戏积累情绪 |
| 单章节奏 | 开头即高潮 | 逐渐推进到高潮 |
| 读者期待 | 下一个精彩事件 | 下一个高潮点 |
