### 五份规划文档及落地情况

1. **OPTIMIZATION_BLUEPRINT.md** — 产品规划层
   - 三层架构（AI工作层/项目状态层/工作台层）
   - 实施路线图（Phase 1: 结构化索引 / Phase 2: 编排层+UI / Phase 3: 高级功能）
   - ✅ 全部落地

2. **STRUCTURED_INDEX_DESIGN.md** — 结构化索引详细设计
   - 5个索引 schema（characters.json/chapters.json/beats.jsonl/locations.json/timeline.jsonl）
   - decisions.jsonl / world_rules.json schema
   - build_index.py（扫描小说项目生成初始索引）
   - incremental_index_update.py（7个命令：update_chapter/add_beat/update_beat/update_character/add_location/add_timeline_event/rebuild）
   - ✅ 全部落地

3. **ORCHESTRATION_LAYER_DESIGN.md** — 编排层详细设计
   - Role-based Workflow Engine（非重型图框架）
   - 5个角色：Supervisor/Planner/Writer/Editor/World
   - 4条核心 Workflow：write_chapter/review_chapter/update_world/query
   - 状态机7态：pending/running/blocked/waiting_human/done/failed/cancelled
   - workflow_state.py（7个命令：start/advance/complete/status/waiting/resume/list）
   - ✅ 全部落地

4. **ROLES_IMPLEMENTATION_DESIGN.md** — 角色落地详细设计
   - 每个角色的标准输入/输出 schema
   - 4条交接链：Supervisor→Planner→Planner→Writer→Writer→Editor→Editor→Supervisor
   - 必须升级给 Supervisor 的情况（Writer/Editor/World各有权限边界）
   - 人类决策点标准（主角死亡/主线变更/世界规则变更等7类）
   - ✅ 全部落地（但还未真正在Agent执行时调用）

5. **UI_WORKBENCH_DESIGN.md** — UI工作台详细设计
   - 8个页面：Dashboard/Chapter Studio/Beat Board/Timeline/World Bible/Decision Center/Reports+Trace/Settings
   - MVP 4页：Dashboard/Chapter Studio/Beat Board/Settings
   - 核心观点：UI不是展示层，是"作者和系统之间的驾驶舱"
   - ⚠️ 设计文档完成，UI本身未实现（Phase 2优先级）

### 架构核心原则（神风大王强调）

1. **通用性**：novel-agent是通用工具，不能绑定具体小说（但MEMORY.md里《末世·毒舌系统》是我自己的项目，分开维护）
2. **AGENT.md是核心**：SKILL.md和CLAUDE.md是薄适配层，不是核心
3. **配置与内容分离**：config/是框架级配置，与具体小说完全解耦
4. **索引优先**：AI先读索引再读正文，降低token成本，提高稳定性
5. **角色分工明确**：不追求炫技的多agent，而是清晰的角色+状态+交接规则
